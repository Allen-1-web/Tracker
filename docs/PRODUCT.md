# Product direction

## Главное сообщение

**Tracker помогает держать ежедневные привычки.**

Всё остальное — вспомогательные модули (цели, питание, напоминания), но не ядро позиционирования.

## Telegram — companion, не второй frontend

Telegram-бот — главная уникальность проекта:

- быстрые отметки привычек и сводки дня;
- напоминания в привычном мессенджере;
- `/report` и digest-уведомления для retention.

Веб-приложение — полноценный UI для настройки, аналитики и детального управления. Бот **не дублирует** весь веб: он дополняет его там, где мессенджер удобнее.

## Admin panel — заморожен

Для MVP и internal ops достаточно:

- SQL и `backend/supabase/scripts/bootstrap-admin.sql.example`;
- Supabase Dashboard;
- минимальный `/admin` (только смена роли user/admin).

**Не развиваем** admin panel дальше: метрики, модерация, bulk-операции — через SQL/Supabase, пока не появится явная пользовательская ценность.

## Приоритеты разработки

| Приоритет | Направление |
|-----------|-------------|
| **P0** | Привычки: отметки, streaks, `/stats` |
| **P1** | Telegram companion: отметки, digest, напоминания |
| **P2** | Retention: weekly/monthly recap в вебе и боте |
| **P3** | Цели, питание — secondary, без расширения scope |
| **—** | Admin panel — только поддержка существующего минимума |

## `/stats` — analytics hub

Страница `/stats` (раздел «Аналитика») — ключевой инструмент retention:

- streaks (идеальные дни + лучший streak по привычкам);
- completion rate за 7 / 30 / 90 дней;
- heatmap активности за год;
- weekly bar chart (12 недель);
- weekly и monthly recap;
- рейтинг лучших и слабых привычек.

Per-habit analytics остаётся на `/habits/[id]`.

## Что не делаем сейчас

- Расширение admin panel.
- Позиционирование как «универсальной productivity-системы».
- Паритет feature set между вебом и Telegram.
